# 数据库配置修复指南

## ❌ 错误信息

```
Failed to publish your Function. Got error: Error 8000022: Invalid database UUID
  (local-development). Check your database UUID and try again.
```

## 🔍 问题原因

`wrangler.jsonc` 中的 `database_id` 设置为 `"local-development"`，这是**本地开发占位符**，不能用于生产部署。

---

## ✅ 解决方案

### 方案 1：创建真实的 D1 数据库（推荐）

如果您需要使用数据库功能（CMS、导航管理等），需要创建真实的 D1 数据库。

#### 步骤 1：创建 D1 数据库

```bash
# 创建生产数据库
npx wrangler d1 create zenava-production
```

**输出示例：**
```
✅ Successfully created DB 'zenava-production'!

[[d1_databases]]
binding = "DB"
database_name = "zenava-production"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"  ← 复制这个 ID
```

#### 步骤 2：更新 wrangler.jsonc

将输出的 `database_id` 复制到 `wrangler.jsonc`：

```jsonc
{
  "d1_databases": [
    {
      "binding": "DB",
      "database_name": "zenava-production",
      "database_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"  // ← 替换为真实的 ID
    }
  ]
}
```

#### 步骤 3：应用数据库迁移

```bash
# 应用迁移到生产数据库
npm run db:migrate:prod

# 如果需要初始化数据
npm run db:seed
```

#### 步骤 4：重新部署

```bash
npm run deploy:prod
```

---

### 方案 2：暂时禁用数据库（快速部署）

如果您暂时不需要数据库功能，可以先注释掉 D1 配置，完成部署。

#### 步骤 1：注释 D1 配置

编辑 `wrangler.jsonc`：

```jsonc
{
  // ... 其他配置 ...
  
  // 暂时禁用 D1 数据库（如果不需要数据库功能）
  // "d1_databases": [
  //   {
  //     "binding": "DB",
  //     "database_name": "zenava-production",
  //     "database_id": "local-development"
  //   }
  // ],
}
```

#### 步骤 2：检查代码是否依赖数据库

如果代码中有使用 `c.env.DB`，需要：

1. **检查代码**：搜索 `c.env.DB` 或 `env.DB`
2. **添加容错处理**：如果数据库不存在，使用默认配置

#### 步骤 3：重新部署

```bash
npm run deploy:prod
```

## 🔍 检查数据库使用情况

### 搜索数据库引用

```bash
# 搜索代码中的数据库使用
grep -r "c.env.DB" src/
grep -r "env.DB" src/
grep -r "DB\." src/
```

### 常见使用场景

- ✅ **CMS 内容管理**：需要数据库
- ✅ **导航菜单管理**：需要数据库（如果使用 CMS）
- ✅ **页脚内容管理**：需要数据库（如果使用 CMS）
- ❌ **纯静态页面**：不需要数据库

## 📋 完整部署流程（使用数据库）

```bash
# 1. 登录 Cloudflare
npx wrangler login

# 2. 创建 D1 数据库
npx wrangler d1 create zenava-production

# 3. 复制 database_id 到 wrangler.jsonc

# 4. 应用数据库迁移
npm run db:migrate:prod

# 5. 初始化数据（可选）
npm run db:seed

# 6. 构建项目
npm run build

# 7. 部署
npm run deploy:prod
```

## 🆘 常见问题

### Q1: 创建数据库失败？

**检查清单：**
- ✅ 是否已登录 Cloudflare（`npx wrangler whoami`）
- ✅ API Token 是否有 D1 权限
- ✅ 账户是否有 D1 配额（免费账户有配额限制）

**解决方案：**
```bash
# 重新登录
npx wrangler login

# 检查账户信息
npx wrangler whoami
```

### Q2: 迁移失败？

**检查清单：**
- ✅ database_id 是否正确
- ✅ 迁移文件是否存在（`migrations/*.sql`）
- ✅ 数据库是否已创建

**解决方案：**
```bash
# 检查数据库列表
npx wrangler d1 list

# 查看数据库信息
npx wrangler d1 info zenava-production

# 重新应用迁移
npm run db:migrate:prod
```

### Q3: 部署后数据库连接失败？

**检查清单：**
- ✅ wrangler.jsonc 中的 database_id 是否正确
- ✅ 数据库是否已创建
- ✅ 迁移是否已应用

**解决方案：**
```bash
# 检查数据库绑定
npx wrangler pages deployment list --project-name zenava-official

# 查看实时日志
npx wrangler tail --project-name zenava-official
```
