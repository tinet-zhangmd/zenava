# 打包方式说明

## 🎯 核心答案

**打包方式基本不变，但运行方式不同！**

---

## 📦 当前打包方式

### 统一构建命令

```bash
npm run build
```

**生成内容：**
- `dist/_worker.js` - Cloudflare Worker 格式的代码
- `dist/assets/` - 静态资源（图片、视频等）
- `dist/static/` - 静态文件（CSS、JS等）
- `dist/_routes.json` - 路由配置

---

## 🔄 两种部署方式的区别

### 方案 A：Nginx + Wrangler

**打包方式：** ✅ **完全相同**
```bash
npm run build  # 生成 dist/ 目录
```

**运行方式：**
```bash
# Wrangler 直接运行 dist/ 目录
npx wrangler pages dev dist --port 3000
```

**特点：**
- ✅ 打包方式不变
- ✅ 使用相同的 `dist/` 目录
- ✅ Wrangler 运行 `_worker.js`

---

### 方案 B：Nginx + Node.js

**打包方式：** ⚠️ **有两种选择**

#### 选择 1：使用相同构建（推荐）

```bash
npm run build  # 仍然使用相同的构建
```

**运行方式：**
```bash
# 使用 server.js 运行，但需要适配代码
node server.js
```

**说明：**
- ✅ 打包命令不变
- ⚠️ 但 `_worker.js` 是 Cloudflare Workers 格式，Node.js 无法直接运行
- ✅ 需要 `server.js` + `src/index-node.tsx` 来适配

#### 选择 2：独立 Node.js 构建（如果完全分离）

```bash
# 需要创建新的构建配置
npm run build:node  # 构建 Node.js 版本
```

**说明：**
- ⚠️ 需要额外的构建配置
- ⚠️ 需要维护两套构建流程
- ❌ 不推荐，除非完全分离两个版本

---

## 💡 推荐方案

### 使用相同的打包方式

**无论选择哪种部署方式，都使用：**

```bash
npm run build
```

**区别只在运行方式：**

| 部署方式 | 打包命令 | 运行命令 |
|---------|---------|---------|
| Wrangler | `npm run build` | `wrangler pages dev dist` |
| Node.js | `npm run build` | `node server.js` |

---

## 📋 实际工作流

### 云效流水线配置

**构建步骤（两种方式都相同）：**
```bash
# 1. 安装依赖
npm install

# 2. 构建项目
npm run build

# 3. 打包部署文件
npm run build:package  # 可选，生成部署包
```

**部署步骤（根据选择不同）：**

**方式 A（Wrangler）：**
```bash
# 在目标服务器上
pm2 start npx --name zenava -- wrangler pages dev dist --port 3000
```

**方式 B（Node.js）：**
```bash
# 在目标服务器上
pm2 start server.js --name zenava
```

---

## ✅ 总结

1. **打包方式：** ✅ **不变** - 都使用 `npm run build`
2. **构建产物：** ✅ **相同** - 都生成 `dist/` 目录
3. **运行方式：** ⚠️ **不同** - Wrangler vs Node.js
4. **代码适配：** ⚠️ **Node.js 需要适配** - 但打包命令不变

---

## 🎯 建议

**对于云效流水线：**

1. **构建步骤统一：**
   ```bash
   npm install
   npm run build
   ```

2. **部署步骤根据环境选择：**
   - 测试环境：可以用 Wrangler（快速）
   - 生产环境：推荐 Node.js（性能好）

3. **打包脚本：**
   ```bash
   npm run build:package  # 生成包含所有文件的部署包
   ```

**关键点：无论选择哪种方式，构建命令都是 `npm run build`！**
