# 图片显示修复说明

## ❌ 问题描述

部署到 Cloudflare Pages 后，`/assets/images/` 下的图片无法显示（显示为破图）。

## 🔍 问题原因

1. **路由配置问题**：`dist/_routes.json` 排除了 `/assets/*` 路径
2. **静态文件服务缺失**：Worker 中没有处理 `/assets/*` 请求的静态文件服务

## ✅ 修复方案

### 1. 在 Worker 中添加静态文件服务

**文件**: `src/index.tsx`

**修改前**:
```typescript
// /assets/* files are in dist/assets/ and should be served by Wrangler automatically
```

**修改后**:
```typescript
// Serve /assets/* files from dist/assets/ directory
app.use('/assets/*', serveStatic({ root: './dist' }))
```

### 2. 更新路由配置

**文件**: `dist/_routes.json`（构建时自动生成）

**修改前**:
```json
{"version":1,"include":["/*"],"exclude":["/static/*","/assets/*"]}
```

**修改后**:
```json
{"version":1,"include":["/*"],"exclude":["/static/*"]}
```

现在 `/assets/*` 请求会被 Worker 处理，而不是被排除。

## 🔧 技术说明

### Cloudflare Pages 静态文件服务机制

1. **自动服务**：Cloudflare Pages 会自动服务 `dist/` 目录下的静态文件
2. **路由配置**：`_routes.json` 控制哪些路径由 Worker 处理，哪些由静态文件服务处理
3. **Worker 处理**：如果路径不在 `exclude` 列表中，请求会被 Worker 处理

### 为什么需要 Worker 处理？

虽然 Cloudflare Pages 可以自动服务静态文件，但在某些情况下（如需要自定义缓存头、访问控制等），通过 Worker 处理更灵活。

### 静态文件路径映射

| 请求路径 | 文件路径 | 说明 |
|---------|---------|------|
| `/assets/images/logo.png` | `dist/assets/images/logo.png` | 图片资源 |
| `/assets/video/codegen.mov` | `dist/assets/video/codegen.mov` | 视频资源 |
| `/static/style.css` | `public/static/style.css` | 静态文件 |
