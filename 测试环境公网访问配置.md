# 测试环境公网访问配置

## 🎯 目标

让测试环境可以通过公网 IP 或域名被任何人访问。

---

## ✅ 当前配置检查

您的配置已经正确设置了 `--ip 0.0.0.0`，这意味着：

```bash
wrangler pages dev dist --ip 0.0.0.0 --port 3000
```

✅ **已绑定到所有网络接口**，允许外部访问

---

## 🔧 完整配置步骤

### 方案 1: 直接访问（简单，适合快速测试）

#### 步骤 1: 启动应用

```bash
# 使用 PM2 启动（推荐）
pm2 start npx --name zenava-test -- wrangler pages dev dist --ip 0.0.0.0 --port 3000

# 或直接运行
npx wrangler pages dev dist --ip 0.0.0.0 --port 3000
```

#### 步骤 2: 配置防火墙

**Ubuntu/Debian:**
```bash
# 开放 3000 端口
sudo ufw allow 3000/tcp
sudo ufw reload
```

**CentOS/RHEL:**
```bash
# 开放 3000 端口
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

**阿里云 ECS:**
1. 登录阿里云控制台
2. 进入 ECS 实例 → 安全组
3. 添加入站规则：
   - 端口：3000
   - 协议：TCP
   - 授权对象：0.0.0.0/0（允许所有 IP）

#### 步骤 3: 获取服务器 IP

```bash
# 查看公网 IP
curl ifconfig.me
# 或
curl ip.sb
```

#### 步骤 4: 访问

```
http://您的服务器IP:3000
```

**示例：**
```
http://123.456.789.0:3000
```

---

### 方案 2: Nginx 反向代理（推荐，更安全）

#### 步骤 1: 启动应用（绑定本地）

```bash
# 绑定到 127.0.0.1（只允许本地访问）
pm2 start npx --name zenava-test -- wrangler pages dev dist --ip 127.0.0.1 --port 3000
```

#### 步骤 2: 配置 Nginx

编辑 `nginx.conf`：

```nginx
server {
    listen 80;
    server_name your-test-domain.com;  # 或使用 IP
    
    # 反向代理到本地应用
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 步骤 3: 应用 Nginx 配置

```bash
sudo cp nginx.conf /etc/nginx/sites-available/zenava-test
sudo ln -s /etc/nginx/sites-available/zenava-test /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 步骤 4: 配置防火墙

```bash
# 只开放 80 端口（HTTP）
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp  # 如果使用 HTTPS
```

#### 步骤 5: 访问

```
http://您的服务器IP
# 或
http://your-test-domain.com
```

## 📋 快速部署脚本

创建 `deploy-test-public.sh`：

```bash
#!/bin/bash
# 测试环境公网访问部署脚本

set -e

PORT=3000
SERVER_IP=$(curl -s ifconfig.me)

echo "🚀 部署测试环境（公网访问）"
echo "服务器 IP: $SERVER_IP"
echo ""

# 1. 构建
echo "🔨 构建项目..."
npm run build

# 2. 停止旧进程
echo "🛑 停止旧进程..."
pm2 stop zenava-test 2>/dev/null || true
pm2 delete zenava-test 2>/dev/null || true

# 3. 启动应用
echo "🚀 启动应用..."
pm2 start npx --name zenava-test -- wrangler pages dev dist --ip 0.0.0.0 --port $PORT

# 4. 配置防火墙（Ubuntu/Debian）
if command -v ufw &> /dev/null; then
    echo "🔥 配置防火墙..."
    sudo ufw allow $PORT/tcp
    sudo ufw reload
fi

# 5. 显示访问地址
echo ""
echo "✅ 部署完成！"
echo ""
echo "📋 访问地址:"
echo "  http://$SERVER_IP:$PORT"
echo "  http://$SERVER_IP:$PORT/zh"
echo ""
echo "📝 管理命令:"
echo "  - 查看状态: pm2 status"
echo "  - 查看日志: pm2 logs zenava-test"
echo "  - 停止应用: pm2 stop zenava-test"
```

## 🔒 安全建议

### 1. 使用 Nginx 反向代理（推荐）

**优点：**
- ✅ 可以配置 SSL/HTTPS
- ✅ 可以配置访问限制
- ✅ 隐藏真实端口
- ✅ 更好的安全控制

### 2. 配置访问限制（可选）

在 Nginx 中限制访问：

```nginx
# 只允许特定 IP 访问
location / {
    allow 192.168.1.0/24;  # 允许内网
    allow 123.456.789.0;   # 允许特定 IP
    deny all;              # 拒绝其他
    
    proxy_pass http://127.0.0.1:3000;
}
```

### 3. 使用 HTTPS（生产环境必须）

```bash
# 使用 Let's Encrypt
sudo certbot --nginx -d your-test-domain.com
```

## 💡 常见问题

### Q: 无法从外部访问？

**检查清单：**
1. ✅ 应用是否绑定到 `0.0.0.0`（不是 `127.0.0.1`）
2. ✅ 防火墙是否开放端口
3. ✅ 安全组是否配置（云服务器）
4. ✅ 应用是否正常运行

### Q: 如何查看访问日志？

```bash
# PM2 日志
pm2 logs zenava-test

# Nginx 日志（如果使用）
sudo tail -f /var/log/nginx/access.log
```

### Q: 如何限制访问？

使用 Nginx 配置 IP 白名单，或使用 Cloudflare 的访问控制。
