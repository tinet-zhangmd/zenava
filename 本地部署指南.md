# 本地部署指南

快速在本地环境部署和测试 Zenava Webapp。

## 🚀 快速开始

### 方法 1: 使用部署脚本（推荐）

```bash
# 运行本地部署脚本
npm run deploy:local

# 或直接运行
./deploy-local.sh
```

脚本会自动：
1. ✅ 检查环境（Node.js、端口）
2. 📦 安装依赖
3. 🔨 构建项目
4. 🖥️ 启动服务器

### 方法 2: 手动步骤

```bash
# 1. 安装依赖
npm install

# 2. 构建项目
npm run build

# 3. 启动服务器
npm run start:local
# 或
npx wrangler pages dev dist --ip 0.0.0.0 --port 3000
```

## 📋 启动方式选择

### 方式 1: 直接运行（前台）

适合：快速测试、开发调试

```bash
npx wrangler pages dev dist --ip 0.0.0.0 --port 3000
```

**特点**：
- ✅ 简单直接
- ✅ 可以看到实时日志
- ❌ 关闭终端会停止服务

### 方式 2: PM2 运行（后台）

适合：长时间运行、生产模拟

```bash
# 安装 PM2（如果未安装）
npm install -g pm2

# 启动
pm2 start npx --name zenava-local -- wrangler pages dev dist --ip 0.0.0.0 --port 3000

# 查看状态
pm2 status

# 查看日志
pm2 logs zenava-local

# 停止
pm2 stop zenava-local
```

**特点**：
- ✅ 后台运行
- ✅ 自动重启
- ✅ 日志管理
- ✅ 适合生产环境模拟

## 🌐 访问地址

启动成功后，访问：

- **本地访问**: http://localhost:3000
- **局域网访问**: http://你的IP:3000

## 🔧 常见问题

### 1. 端口被占用

```bash
# 查看占用端口的进程
lsof -i :3000

# 停止进程
kill -9 <PID>
# 或
fuser -k 3000/tcp
```

### 2. 构建失败

```bash
# 清理并重新构建
rm -rf dist node_modules
npm install
npm run build
```

### 3. PM2 命令不存在

```bash
# 安装 PM2
npm install -g pm2
```

### 4. 查看日志

```bash
# 如果使用 PM2
pm2 logs zenava-local

# 如果直接运行，日志会直接显示在终端
```
