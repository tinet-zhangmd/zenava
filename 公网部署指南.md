# Zenava 公网部署指南

## 🎯 部署方案选择

根据您的需求，有三种部署方案：

| 方案 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **Cloudflare Pages** | ✅ 免费<br>✅ 全球CDN<br>✅ 自动HTTPS<br>✅ 无需服务器 | ❌ 需要Cloudflare账号<br>❌ 数据库需使用D1 | **推荐：快速上线** |
| **自建服务器 + Nginx** | ✅ 完全控制<br>✅ 可使用SQLite<br>✅ 灵活配置 | ❌ 需要服务器<br>❌ 需维护 | 有服务器资源 |
| **Node.js 直接运行** | ✅ 最简单<br>✅ 快速测试 | ❌ 不适合生产<br>❌ 无HTTPS | 仅测试环境 |

---

## 🚀 方案 1：Cloudflare Pages（推荐）

### 前置要求
- ✅ Cloudflare 账号（免费注册：https://dash.cloudflare.com）
- ✅ 已安装 Node.js 和 npm
- ✅ 项目已构建（`npm run build`）

### 步骤 1：登录 Cloudflare

```bash
# 在项目目录下执行
npx wrangler login

# 或使用 API Token（推荐）
# 1. 访问 https://dash.cloudflare.com/profile/api-tokens
# 2. 创建 Token，选择 "Edit Cloudflare Workers" 权限
# 3. 设置环境变量
export CLOUDFLARE_API_TOKEN="your-api-token-here"
```

### 步骤 2：创建 D1 数据库（如果使用数据库）

```bash
# 创建生产数据库
npx wrangler d1 create zenava-production

# 复制输出的 database_id，更新到 wrangler.jsonc
# 例如：
# "database_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

### 步骤 3：应用数据库迁移

```bash
# 应用迁移到生产数据库
npm run db:migrate:prod

# 如果需要初始化数据
npm run db:seed
```

### 步骤 4：部署到 Cloudflare Pages

```bash
# 方式1：使用部署脚本（推荐）
npm run deploy:prod

# 方式2：手动部署
npm run build
npx wrangler pages deploy dist --project-name zenava-official
```

### 步骤 5：配置自定义域名（可选）

1. 登录 Cloudflare Dashboard
2. 进入 Pages 项目
3. 点击 "Custom domains"
4. 添加您的域名
5. 按照提示配置 DNS 记录

### ✅ 部署完成

访问地址：
- **默认地址**: `https://zenava-official.pages.dev`
- **自定义域名**: `https://your-domain.com`

## 🖥️ 方案 2：自建服务器 + Nginx

### 前置要求
- ✅ 云服务器（阿里云/腾讯云/AWS等）
- ✅ 域名（已备案，如使用国内服务器）
- ✅ SSH 访问权限

### 步骤 1：准备服务器

```bash
# 1. 连接到服务器
ssh root@your-server-ip

# 2. 更新系统
sudo apt update && sudo apt upgrade -y

# 3. 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 4. 安装 PM2
sudo npm install -g pm2

# 5. 安装 Nginx
sudo apt install -y nginx
```

### 步骤 2：上传项目

```bash
# 在服务器上创建项目目录
sudo mkdir -p /var/www/zenava
sudo chown -R $USER:$USER /var/www/zenava

# 在本地执行（上传项目文件）
scp -r . user@your-server-ip:/var/www/zenava

# 或在服务器上使用 Git
cd /var/www/zenava
git clone your-repo-url .
```

### 步骤 3：安装依赖并构建

```bash
cd /var/www/zenava

# 安装依赖
npm install

# 构建项目
npm run build

# 创建数据目录
mkdir -p data
```

### 步骤 4：配置 Nginx

```bash
# 复制配置文件
sudo cp nginx.conf /etc/nginx/sites-available/zenava

# 编辑配置（修改域名）
sudo nano /etc/nginx/sites-available/zenava

# 启用站点
sudo ln -s /etc/nginx/sites-available/zenava /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 步骤 5：启动应用

```bash
cd /var/www/zenava

# 使用 Node.js 服务器（推荐）
pm2 start server.js --name zenava-webapp

# 或使用 Wrangler（如果使用 D1 数据库）
pm2 start npx --name zenava-webapp -- wrangler pages dev dist --ip 127.0.0.1 --port 3000

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

### 步骤 6：配置 SSL（HTTPS）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

### ✅ 部署完成

访问地址：
- **HTTP**: `http://your-domain.com`
- **HTTPS**: `https://your-domain.com`

## 🧪 方案 3：Node.js 直接运行（仅测试）

### 快速启动（公网访问）

```bash
# 1. 构建项目
npm run build

# 2. 启动服务器（绑定到所有网络接口）
npm run start:node

# 或使用 PM2
pm2 start server.js --name zenava-test

# 3. 配置防火墙（开放 3000 端口）
sudo ufw allow 3000/tcp
```

### 访问地址

```
http://your-server-ip:3000
```

⚠️ **注意**：此方案仅适用于测试，生产环境请使用方案 1 或 2。

## 🔧 常见问题

### Q1: Cloudflare Pages 部署失败？

**检查清单：**
1. ✅ 是否已登录 Cloudflare（`npx wrangler whoami`）
2. ✅ API Token 是否正确
3. ✅ 项目名称是否唯一
4. ✅ 构建是否成功

**解决方案：**
```bash
# 重新登录
npx wrangler login

# 检查项目列表
npx wrangler pages project list

# 查看部署日志
npx wrangler pages deployment list --project-name zenava-official
```

### Q2: 服务器部署后无法访问？

**检查清单：**
1. ✅ 防火墙是否开放端口（80/443/3000）
2. ✅ 安全组是否配置（云服务器）
3. ✅ 应用是否正常运行（`pm2 status`）
4. ✅ Nginx 是否正常运行（`sudo systemctl status nginx`）

**解决方案：**
```bash
# 检查端口监听
sudo netstat -tlnp | grep 3000

# 检查防火墙
sudo ufw status

# 检查应用日志
pm2 logs zenava-webapp

# 检查 Nginx 日志
sudo tail -f /var/log/nginx/error.log
```

### Q3: 数据库连接失败？

**检查清单：**
1. ✅ D1 数据库 ID 是否正确（Cloudflare）
2. ✅ 数据库迁移是否已应用
3. ✅ 数据库文件权限是否正确（SQLite）

**解决方案：**
```bash
# Cloudflare D1
npx wrangler d1 list
npx wrangler d1 migrations apply zenava-production

# SQLite
ls -la data/zenava.db
chmod 664 data/zenava.db
```

## 🎯 推荐方案

**如果您是第一次部署，推荐使用：**
